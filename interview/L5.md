123. 无窗口 GROUP BY 的 SQL 为什么产生回撤流？下游 upsert-kafka 与普通 kafka 怎么落地？

**参考答案：** 答案：更新聚合需要撤回旧值再发新值（retract/upsert 语义）。upsert-kafka 以主键覆盖；普通 append-only kafka 不能表达撤回，需 changelog 序列化或换 sink。见 docs/05-sql。

124. Regular / Interval / Temporal Join 状态规模分别由什么决定？

**参考答案：** 答案：Regular：双流历史无界，状态易爆。Interval：由时间窗口界限制。Temporal：维表版本/Lookup 缓存策略决定。见 docs/05-sql。

125. COUNT(DISTINCT) 在流上为何昂贵？三种降本？

**参考答案：** 答案：需记住见过的值。降本：HLL 草图、预聚合下推 OLAP、放大窗口粒度/抽样。精确去重必须论证状态上界。见 best-practice #5。

126. SQL 作业怎么设置状态 TTL？对正确性影响？

**参考答案：** 答案：`table.exec.state.ttl`。TTL 过短→维表/去重「忘却」导致重复或错连；过长→状态膨胀。必须业务论证。见 docs/05-sql。

127. Materialized Table 解决什么问题？与定时批差异？

**参考答案：** 答案：声明式物化与持续刷新，缩短「批调度脚本」与「流结果表」鸿沟；本质仍是引擎维护新鲜度，而非 crontab 盲跑。见 docs/05-sql。

128. Changelog 模式 ALL / UPSERT 对 sink 约束？

**参考答案：** 答案：ALL 含 +/-；UPSERT 靠主键。Sink 必须能消化对应语义，否则结果错。见 docs/05-sql。

129. Lookup Join 缓存过期策略怎么选？

**参考答案：** 答案：维表变更频率 vs 命中率。过短打爆 DB；过长读到旧维表。监控旁路延迟。见 docs/07-connectors。

130. Flink SQL 与 DataStream 混用的边界？

**参考答案：** 答案：toChangelogStream/fromDataStream 需对齐类型与时区；uid 与并行度仍要管。适合 SQL 做管道、DS 做特殊算子。

131. MiniBatch 聚合的收益与风险？

**参考答案：** 答案：降状态访问频率提吞吐；增大延迟、语义需理解。适合高基数组。见 docs/05-sql。

132. Over 窗口与 Group 窗口差异？

**参考答案：** 答案：Over 按行帧计算，状态按 key+帧；Group 是窗口算子。注意 SQL 写法对应的状态含义。

133. Watermark 在 SQL 里如何定义？

**参考答案：** 答案：DDL WATERMARK FOR col AS expr。错误表达式=错误触发。见 docs/05-sql。

134. 主键语义缺失会导致什么？

**参考答案：** 答案：upsert sink 拒写或结果重复；回撤无法对齐。建模先定主键。

135. 时区在 SQL 窗口的坑？

**参考答案：** 答案：TIMESTAMP_LTZ vs TIMESTAMP；窗口边界按本地日切需显式时区，否则「天窗」错位。

136. SQL 作业如何做单元级验证（本仓库）？

**参考答案：** 答案：案例 e0x + 项目 verify 脚本对照 CH/Kafka；禁止只 explain 不跑。见 examples、projects。

137. TopN 流式实现要点？

**参考答案：** 答案：追加/回撤更新排名；状态按分区键+排序键；注意结果回撤风暴。见 docs/05-sql。

138. CDC 入 SQL 表后如何保证主键更新？

**参考答案：** 答案：源表正确 PRIMARY KEY、changelog 解释正确、下游 upsert 支持。见 docs/08-cdc。

139. Plan 解释器里能看到什么排障信息？

**参考答案：** 答案：算子链、changelog mode、upsert key。用于确认是否意外 retract。

140. SQL Client 与 jar 提交差异？

**参考答案：** 答案：Client 适合交互；生产用 jar/SQL gateway/Operator 声明。本仓库以 jar+Makefile 为主。

141. 双流 join 超时清理参数？

**参考答案：** 答案：idle state retention / TTL / interval 界。不设则状态无限涨。

142. 为什么「先 SQL 建模再下推 DS」是好路径？

**参考答案：** 答案：先保证关系语义可测，再对瓶颈算子定制；反之过早微优化难回归。

143. DDL 与 DML 分离的好处？

**参考答案：** 答案：环境差异只换 DDL 连接属性；DML 可复用。

145. upsert 主键变更 equates 删旧插新？

148. 回撤风暴的业务症状？
