# L8+ · AI 专项（p01 / Agents / RAG）

> 共 30 题。先自答，再展开「参考答案」。链回 `docs/`/`ai/`/`projects/` 仅作延伸，不替代本题答案正文。

201. 什么样的 AI 场景该进 Flink？三条判据。

**参考答案：** 答案：1) 要与流状态/窗口/CEP 同源处理 2) 延迟与吞吐需背压可控 3) 必须可降级（无模型也能出规则结果）。纯离线批训不必进 Flink。见 ai/ 第 0 节、p01 ADR。

202. ML_PREDICT / LLM 调用超时、限流、降级如何设计？

**参考答案：** 答案：Async I/O + 超时；BudgetGate 限流（p01）；失败打 `ai_source=DEGRADED` 走规则；指标 `ai_timeouts`/`ai_degrades`。见 p01、ai/03。

203. Flink Agents Action 级 exactly-once 与工具副作用？

**参考答案：** 答案：Action 与 checkpoint 协同提交；外部工具副作用需幂等或补偿。真实可用性以本机 Agents 版本验证为准（Preview）。见 ai/09。

204. 流式 RAG 与常规 RAG 差异？新鲜度如何量化？

**参考答案：** 答案：流式侧重持续更新索引/特征与在线检索延迟；新鲜度可用「事件时间→可检索」滞后分钟数 + 召回命中率。见 ai/05。

205. 为何 BudgetGate 必须放在 Async 外呼之前？

**参考答案：** 答案：之后只能事后打标，外呼已发生，打爆 Ollama。p01 在 Async 前 Side Output 短路并 `budget_trips++`。见 p01 README 面试题。

206. Guardrail 阻断后为何仍要落库？

**参考答案：** 答案：审计与可观测：`ai_source=BLOCKED` 截断原文仍可分析误报；静默丢弃无法追责。见 GuardrailFunction。

207. AI 指标在 Prometheus 的真实名字是什么？

**参考答案：** 答案：`flink_taskmanager_job_task_operator_p01_ai_calls`、`_ai_timeouts`、`_ai_degrades`、`_budget_trips`、`_guardrail_blocks`（MetricGroup `p01`）。看板 `ai-cost.json`。见 monitoring。

208. 无 Ollama 时 p01 默认门禁为何仍绿？

**参考答案：** 答案：`--ai.enabled=false` 规则路径，`ai_source=DISABLED`，零外呼；verify 查 CH 规则标签。见 DEGRADE-CHECKLIST。

209. 向量检索（Milvus）失败时架构如何降级？

**参考答案：** 答案：跳过 RAG 增强，回退规则/关键词；记录降级指标；不阻断主日志管道。见 ai-profile、p01 纪律。

210. 提示词注入类风险在流式管道如何挡？

**参考答案：** 答案：输出 Guardrail 关键词/模式阻断；不把模型输出当代码执行；权限最小化。见 Guardrail、threat model。

211. token/成本治理没有独立计费网关时如何做？

**参考答案：** 答案：用调用次数预算（BudgetGate）+ 超时/降级计数近似成本；Grafana ai-cost 观察。不新拉未登记计费栈。见 D-10/D-11。

212. 异步 LLM 与 checkpoint 交互的注意点？

**参考答案：** 答案：AsyncWait 超时与重试不影响 JM；进行中请求需在故障后可重复且幂等。见 AsyncDataStream 用法、p01。

213. 为何禁止算子内同步 HTTP 调模型？

**参考答案：** 答案：阻塞 Task 线程，吞吐被并行度×(1000/延迟)钉死，并拖垮 checkpoint。见 best-practice #4。

214. AI 作业的「正确性」如何定义？

**参考答案：** 答案：规则标签永远可用；AI 风险分是增强字段；降级不丢主数据。验收以 CH 字段与 counter 为准。见 p01 verify。

215. 流式特征与训练闭环的最小可行架构？

**参考答案：** 答案：行为事件→Flink 特征→Redis/CH；训练离线拉特征；在线只读。避免在线作业内训大模型。见 p02 方向。

216. 模型切换如何不影响 uid/状态？

**参考答案：** 答案：模型名作配置而非算子 uid；状态只存业务特征，不存模型权重。

217. 评估集与线上一致性如何保证？

**参考答案：** 答案：同一解析/护栏逻辑复用；离线回放 Kafka 段对比 `ai_risk` 分布。见 ai eval 思路。

218. Agents Preview 坐标漂移怎么办？

**参考答案：** 答案：遵循版本矩阵；不可用则文档降级路径，不挡主门禁。见 CLAUDE.md stack。

219. 多模态日志（文本+向量）管道的背压点？

**参考答案：** 答案：嵌入推理与向量写入；需独立预算与批量。指标分轨观察。

220. 用三个指标讲述一次 AI 故障值班故事。

**参考答案：** 答案：例：`budget_trips` 飙升→达预算；`ai_timeouts` 升→模型慢；`guardrail_blocks` 升→提示词/敏感内容。结合 `numRestarts=0` 说明降级生效未拖垮作业。见 ai-cost 看板。

221. DEGRADED 与 DISABLED 语义差？

**参考答案：** 答案：DISABLED=从未开 AI；DEGRADED=尝试后失败/熔断。CH 字段可区分。

222. 为何 AI 面板用 Counter 而非假造 token 美元？

**参考答案：** 答案：本环境真实暴露为 Flink Counter；美元需价目表，未登记则用调用次数代理。D-10。

223. RAG 检索空结果时？

**参考答案：** 答案：回退纯规则/LLM 短提示；记录空召回率。

224. 提示词版本如何管理？

**参考答案：** 答案：配置中心/作业参数版本号；A/B 用旁路指标。

225. 安全：日志进模型前要做什么？

**参考答案：** 答案：脱敏、护栏、预算；禁止原样送密钥。见 DEGRADE/GUARD。

226. p01 验证脚本 verify vs verify-ai？

**参考答案：** 答案：verify 默认规则绿；verify-ai 可选 Ollama。门禁不绑 GPU。

227. 成本面板为何对照 checkpoint/restarts？

**参考答案：** 答案：证明熔断时作业仍健康（重启不增），降级策略有效。

228. Milvus 未启动时的可观察信号？

**参考答案：** 答案：连接失败日志+降级路径指标；主 verify 仍可通过。

229. 规则与模型不一致时以谁为准？

**参考答案：** 答案：安全/合规以规则与护栏为准；模型仅建议。

230. budget.max-ai-calls 如何定？

**参考答案：** 答案：压测下 Ollama QPS×安全系数；超则 trips 升但仍绿。
